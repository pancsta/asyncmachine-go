> [!NOTE]
> **asyncmachine-go** is a batteries-included graph control flow library (AOP, actor model, state-machine).

A simple example of WASM interop and communication between two distributed async state machines.

Source files:

- shared [wasm_example.go](/examples/wasm/wasm_example.go)
- client [wasm_client_machs.go](/examples/wasm/client/wasm_client_machs.go)
- server [wasm_server.go](/examples/wasm/server/wasm_server.go)

Actors (server):

- `server-foo` main state machine
  - RPC server (WS)
- aRPC client (WS / TCP)
  - `rc-server-bar` network mach

Actors (browser):

- `browser-bar` UI state machine
  - RPC server (WS / TCP over WS)
- aRPC client (WS)
  - `rnm-browser-foo` network mach
  - `browser-foo` net handler mach

## Implementation

`net/rpc`, [cenkalti/rpc2](https://github.com/cenkalti/rpc2), and [coder/websocket](https://github.com/coder/websocket)
with TCP-over-WebSocket bridged using [`/tools/cmd/am-relay`](/tools/cmd/am-relay/README.md).

![logical diagram](https://pancsta.github.io/assets/asyncmachine-go/diagrams/example-wasm.d2.dark.svg)

## Debugging

Both `server-foo` and `browser-bar` connect to a single [`am-dbg`](/tools/cmd/am-dbg/README.md) instance running on
the server and can be stepped through, with context from all the other machines.

Both `server-foo` and `browser-bar` expose REPLs.

![am-dbg WASM](https://pancsta.github.io/assets/asyncmachine-go/examples/wasm-am-dbg.png)

Interactively use the [TUI debugger](/tools/cmd/am-dbg) with data pre-generated by this example:

```bash
go run github.com/pancsta/asyncmachine-go/tools/cmd/am-dbg@latest \
  --import-data https://assets.asyncmachine.dev/am-dbg-exports/wasm.gob.br \
  mach://server-foo
```

### Extended Network Diagram

The extended machine diagram additionally contains RPC clients and servers (aka noise).

```bash
# generate with .env
AM_RPC_LOG_SERVER=1
AM_RPC_LOG_CLIENT=1
AM_RPC_LOG_MUX=1
AM_RPC_DBG=1

# render
am-vis --bird render-dump am-dbg-dump.gob.br
```
