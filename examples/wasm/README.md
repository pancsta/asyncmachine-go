# <img src="https://pancsta.github.io/assets/asyncmachine-go/logo.png" height="25"/> /examples/WASM

[`cd /`](/README.md)

> [!NOTE]
> **asyncmachine-go** is a pathless control-flow graph with a consensus (AOP, actor model, state-machine).

This is a simple example of WASM interop and communication between two distributed async state machines
(`foo` and `bar`). The complete WASM support allows having a single state-machine which is fully (or selectively)
distributed across _n_ servers and _m_ browsers, using efficient diff-based synchronization.

## Running

1. `git clone https://github.com/pancsta/asyncmachine-go.git`
2. `cd asyncmachine-go/examples/wasm`
3. `task start`
  - no `task`? https://taskfile.dev/docs/installation

## Implementation

[cenkalti/rpc2](https://github.com/cenkalti/rpc2) (fork of `net/rpc`), and [coder/websocket](https://github.com/coder/websocket)
with TCP-over-WebSocket bridged using [`/tools/cmd/am-relay`](/tools/cmd/am-relay/README.md). The connection between
`foo` and `bar` machines is
full-duplex, meaning each side is both a client and a server for the other one. This allows for mutations passing both
sides.

![logical diagram](https://pancsta.github.io/assets/asyncmachine-go/diagrams/example-wasm.d2.dark.svg)

> [!NOTE]
> Blue - real state machine
> Blue+orange - real state machine and state source
> Green - network state machine (mirror)
> Grey - regular struct with methods

## Debugging

Both `server-foo` and `browser-bar` connect to a single [`am-dbg`](/tools/cmd/am-dbg/README.md) instance running on
the server and can be stepped through, with context from all the other machines.

Both `server-foo` and `browser-bar` expose REPLs.

![am-dbg WASM](https://pancsta.github.io/assets/asyncmachine-go/examples/wasm-am-dbg.png?74326487432)

Interactively use the [TUI debugger](/tools/cmd/am-dbg/README.md) with data pre-generated by this example:

```bash
go run github.com/pancsta/asyncmachine-go/tools/cmd/am-dbg@latest \
  --import-data https://assets.asyncmachine.dev/am-dbg-exports/wasm.gob.br \
  mach://server-foo
```

am-vis --bird render-dump am-dbg-dump.gob.br

### Extended Network Diagram

The extended machine diagram additionally contains RPC clients and servers (aka noise).

```bash
# generate with .env
AM_RPC_LOG_SERVER=1
AM_RPC_LOG_CLIENT=1
AM_RPC_LOG_MUX=1
AM_RPC_DBG=1

# render
am-vis --bird render-dump am-dbg-dump.gob.br
```
